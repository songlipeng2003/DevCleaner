## 二、二期功能规划

### 2.1 智能清理引擎（优先级：⭐⭐⭐⭐⭐）

#### 2.1.1 多维度清理策略

| 策略 | 说明 | 用户价值 |
|------|------|----------|
| **按时间清理** | 清理 N 天前的缓存 | 保留近期有用缓存 |
| **按大小清理** | 只清理超过阈值的大缓存 | 优先清理高价值空间 |
| **选择性清理** | 文件级勾选清理 | 精确控制清理范围 |
| **安全清理** | 仅清理确定安全的文件 | 避免误删重要数据 |
| **深度清理** | 清理所有可清理缓存 | 最大化释放空间 |

#### 2.1.2 清理预览增强

```typescript
interface CleanPreview {
  toolId: string
  toolName: string
  paths: PreviewPath[]
  totalSize: number
  riskLevel: 'safe' | 'moderate' | 'careful'
  recommendations: string[]
}

interface PreviewPath {
  path: string
  files: PreviewFile[]
  size: number
  oldestFile: Date
  newestFile: Date
}

interface PreviewFile {
  name: string
  path: string
  size: number
  modified: Date
  isSafe: boolean
  reason?: string
}
```

#### 2.1.3 清理前备份（安全机制）

- **可选备份**：清理前压缩备份到指定目录
- **撤销能力**：7 天内可撤销清理操作
- **操作日志**：记录所有清理操作的详细信息

### 2.2 项目级清理（优先级：⭐⭐⭐⭐⭐）

#### 2.2.1 项目扫描

扫描用户指定目录下的所有项目：

| 类型 | 目录/文件 | 可清理 |
|------|-----------|--------|
| Node.js | `node_modules/` | ✅ 建议清理 |
| Python | `__pycache__/`、`.venv/` | ✅ 建议清理 |
| Go | `vendor/` | ⚠️ 谨慎清理 |
| Rust | `target/` | ✅ 建议清理 |
| .NET | `bin/`、`obj/` | ✅ 建议清理 |
| Ruby | `tmp/cache/`、`log/` | ✅ 建议清理 |

#### 2.2.2 项目清理视图

```
项目目录扫描
├── ~/Projects/
│   ├── my-app/              [3.2 GB]
│   │   ├── node_modules/   [2.1 GB] ← 建议清理
│   │   ├── dist/           [500 MB] ← 谨慎
│   │   └── .next/          [600 MB] ← 可清理
│   ├── api-server/         [1.5 GB]
│   │   ├── target/         [1.2 GB] ← 建议清理
│   │   └── vendor/         [300 MB] ← 谨慎
│   └── data-science/       [800 MB]
│       └── __pycache__/    [50 MB]  ← 建议清理
```

#### 2.6.1 定时自动清理

| 配置项 | 说明 |
|--------|------|

interface AdvancedSettings {
  // 清理规则

    preserveRecent: number      // 保留最近 N 天的缓存

    useTrashBin: boolean        // 移至垃圾桶而非永久删除

  // 扫描配置

  // 性能配置

}
